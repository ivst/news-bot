## Git Branch Naming

Use semantic branch prefixes that match the intent of the change.

Recommended patterns:

- `feat/<short-description>` - new functionality
- `fix/<short-description>` - bug fix
- `refactor/<short-description>` - code restructuring without behavior change
- `chore/<short-description>` - maintenance, tooling, infra
- `docs/<short-description>` - documentation updates
- `test/<short-description>` - tests only

Examples:

- `feat/llm-topic-filter`
- `fix/vk-daily-limit-handling`
- `docs/readme-active-hours`

Keep names short, lowercase, and hyphen-separated.
